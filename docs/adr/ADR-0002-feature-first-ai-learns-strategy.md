# ADR-0002：采用 Feature First，由 AI 学习策略

- 状态：Accepted
- 日期：2026-07-13

## 背景

传统量化通常由人设计完整策略，例如明确规定均线金叉买入、死叉卖出，再由程序执行和回测。

这种方式可解释、易实现，但当因子数量增加、关系变得非线性时，人工组合策略的成本会快速上升，也容易受到主观偏差限制。

本项目希望利用机器学习，让模型自动学习多个因子与未来收益、波动和风险之间的关系。

## 决策

采用 Feature First 的方法：

```text
Human
  -> Problem Definition
  -> Feature Design
  -> Label Design
  -> Risk Constraints

AI Model
  -> Learn Feature Relationships
  -> Produce Predictions
  -> Support Decisions
```

人不再主要负责手工编写所有策略组合，而是负责：

- 明确预测目标
- 设计和验证因子
- 构造训练标签
- 防止数据泄漏
- 设置风险边界
- 评估模型稳定性
- 解释模型输出

AI 负责学习因子之间的权重、交互和非线性关系。

## 预测目标

模型不应只预测涨跌二分类，而应支持多目标输出：

- 未来 1 日上涨概率
- 未来 5 日上涨概率
- 未来 20 日上涨概率
- 未来收益率
- 未来波动率
- 最大回撤
- 风险等级
- 预测置信度

多个模型的结果可以由 Decision Engine 综合。

## 新闻因子的处理

新闻不作为单一情绪分数直接输入。

一条新闻应拆解为多个 Feature，例如：

- 情绪
- 重要性
- 可信度
- 事件类型
- 受影响实体
- 影响方向
- 影响范围
- 时间衰减
- 历史相似事件

同时保存每个 Feature 的来源和生成原因。

## 理由

1. 机器学习更适合处理大量表格型因子和非线性关系。
2. 人可以把精力放在数据质量、因子逻辑和风险控制上。
3. 模型可以快速比较不同因子组合。
4. 同一套 Feature Store 可以支持不同模型。
5. 多目标预测比单一涨跌信号更适合真实决策。

## 风险

- 过拟合历史数据
- 数据泄漏和未来函数
- 标签设计不合理
- 因子高度相关但被误认为多个独立证据
- 市场环境变化导致模型失效
- 模型结果难以解释

## 风险控制

必须采用：

- 时间序列训练、验证和测试切分
- Walk-forward Validation
- Out-of-sample Test
- 交易成本和滑点模拟
- 因子相关性检查
- 特征重要性和贡献分析
- 不同市场环境下的稳定性测试
- 模型版本、数据版本和决策记录

## 模型选择

第一阶段优先选择适合结构化表格数据的中小型模型：

- Logistic Regression
- Random Forest
- XGBoost
- LightGBM

不以参数量作为模型价值标准。

只有当数据规模、任务复杂度和验证结果证明有必要时，再引入更复杂的深度学习模型。

## 影响

### 正面影响

- 降低人工组合大量策略的成本。
- 支持多因子和非线性关系。
- Feature、模型和决策层边界清晰。
- 模型可以替换，数据和因子资产可以长期保留。

### 负面影响

- 对数据质量、标签和验证流程要求更高。
- 模型可能学到历史噪声。
- 必须额外建设可解释性和数据血缘能力。

## 结论

本项目采用“人设计问题、因子和约束，AI 学习关系和策略”的协作模式。

AI 不取代人的判断责任。模型输出是概率和风险信息，最终决策仍需经过回测、风险控制和可解释性验证。
